# PEA-CON Event Semantic Governance Update

ระบบสำหรับลูกค้าสื่อสารรายสำคัญใช้ strict semantic gate ก่อนสร้างข้อมูลจริงสำหรับประเมิน ETR โดยรับเฉพาะสถานะไฟดับหรือไฟกลับที่มาจาก field แบบมีโครงสร้างและอยู่ใน allowlist เท่านั้น รหัส `AC_MAIN_FAIL` ใช้เป็นหลักฐานไฟดับได้ แต่ระบบจะไม่เดาไฟกลับจากข้อความสาเหตุ หากยังไม่พบสัญญาณ RESTORE ที่ตรวจสอบได้ เหตุการณ์จะคงอยู่ใน review และไม่ถูกนำไป train คำนวณ MAE หรือใช้สนับสนุนคำกล่าวอ้างด้านความแม่นยำ ระบบยังทำงานใน shadow mode และ `production_send=blocked` เพื่อควบคุมความเสี่ยงของโมเดลและการสื่อสารกับลูกค้า
